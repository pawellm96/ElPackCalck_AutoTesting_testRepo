# Инструкция по загрузке проекта в Git репозиторий

## Шаг 1: Создайте репозиторий на GitHub/GitLab

1. Зайдите на GitHub (github.com) или GitLab (gitlab.com)
2. Создайте новый репозиторий (New Repository)
3. **НЕ** инициализируйте его с README, .gitignore или лицензией (у нас уже все есть)
4. Скопируйте URL репозитория (например: `https://github.com/ваш-username/ElPackCalc.git`)

## Шаг 2: Выполните первый коммит

Проект уже инициализирован и файлы подготовлены. Выполните:

```bash
# Создать первый коммит
git commit -m "Initial commit: ElPackCalc with tests and CI/CD"

# Добавить удаленный репозиторий (замените URL на ваш)
git remote add origin https://github.com/ваш-username/ElPackCalc.git

# Переименовать ветку в main (если нужно)
git branch -M main

# Отправить код в репозиторий
git push -u origin main
```

## Шаг 3: Проверьте CI/CD

После загрузки кода:

1. Зайдите в раздел "Actions" вашего репозитория
2. Должен автоматически запуститься workflow "CI"
3. Проверьте, что сборка и тесты прошли успешно

## Дальнейшая работа:

```bash
# Добавить изменения
git add .

# Создать коммит
git commit -m "Описание изменений"

# Отправить в репозиторий
git push
```

## Примечания:

- Тесты используют моки (Moq) и рефлексию для тестирования без запуска Revit
- CI/CD автоматически запускается при каждом push
- Результаты тестов сохраняются как артефакты в GitHub Actions
